# 모듈과라이브러리
---

# 🧩 **핵심 차이 한 줄 요약**
- **모듈(Module)**: 프로그램을 구성하는 **작은 단위(파일/기능 단위)**  
- **라이브러리(Library)**: 다른 프로그램에서 가져다 쓰라고 만든 **재사용 가능한 기능 묶음**

---

# 🧱 1) 모듈(Module)이란?

### ✔ 프로그램을 구성하는 “부품”
모듈은 **프로그램 내부를 기능별로 나눈 작은 단위**야.  
하나의 파일이 될 수도 있고, 특정 기능을 담당하는 코드 묶음일 수도 있어.

### ✔ 특징
- 프로그램 내부 구조를 나누기 위한 단위  
- 독립적으로 동작하는 기능 단위  
- 다른 모듈과 조합되어 하나의 프로그램을 구성  
- 언어마다 모듈의 형태가 다름  
  - Python: `.py` 파일  
  - Java: 패키지/모듈 시스템  
  - JavaScript: ES Module (`import/export`)  

### ✔ 예시
- `math.py` (파이썬)  
- `user-service.js` (Node.js)  
- `com.myapp.utils` (Java 패키지)

즉, **모듈은 프로그램을 구성하는 내부 요소**야.

---

# 📚 2) 라이브러리(Library)란?

### ✔ 다른 프로그램에서 가져다 쓰는 “기능 모음집”
라이브러리는 **특정 기능을 제공하는 코드 묶음**으로,  
개발자가 직접 만들지 않고 가져다 쓰는 외부 도구야.

### ✔ 특징
- 재사용 가능한 기능 제공  
- 독립적으로 배포됨  
- 프로그램이 필요할 때 호출해서 사용  
- 개발자가 직접 구현할 필요 없음  

### ✔ 예시
- jQuery (JavaScript)  
- NumPy (Python)  
- Spring Framework (Java)  
- React (UI 라이브러리)  

즉, **라이브러리는 외부에서 가져와 사용하는 기능 패키지**야.

---

# 🔍 **모듈 vs 라이브러리 비교**

| 구분 | 모듈(Module) | 라이브러리(Library) |
|------|--------------|----------------------|
| 목적 | 프로그램 내부 구조화 | 재사용 가능한 기능 제공 |
| 위치 | 프로젝트 내부 | 외부에서 가져옴 |
| 크기 | 작음(파일/기능 단위) | 큼(기능 묶음) |
| 사용 방식 | import/export | import 또는 dependency 추가 |
| 예 | Python 파일, JS 모듈 | NumPy, Lodash, Spring |

---

# 🎯 쉽게 비유하면

- **모듈 = 집 안의 방**  
  → 집(프로그램)을 기능별로 나누는 구조

- **라이브러리 = 가구/가전**  
  → 집에 가져다 놓고 사용하는 외부 도구

---

# 🧠 한 단계 더 깊게: 프레임워크와의 차이

참고로 개발에서 자주 나오는 개념:

- **라이브러리**: 내가 필요할 때 가져다 씀  
- **프레임워크**: 프레임워크가 나의 코드를 호출함 (제어의 역전)
