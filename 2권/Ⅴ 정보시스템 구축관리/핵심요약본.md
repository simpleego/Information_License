# 정보시스템 구축 관리
---

# 📌 **1. 소프트웨어 개발 방법론 – 무조건 출제**
## ✔ 폭포수 모델  
- 단계별 순차 진행  
- 요구사항 변경 어려움

## ✔ 프로토타이핑  
- 시제품 먼저 제작  
- 사용자 요구 반영 쉬움

## ✔ 나선형 모델  
- 위험 분석 중심  
- 반복 + 점진적 개발

## ✔ 애자일(Agile)  
- 고객 참여  
- 반복적 개발  
- 스크럼(Scrum), XP

---

# 📌 **2. 비용산정 기법 – 매년 나옴**
## ✔ 하향식  
- 전문가 판단  
- 델파이 기법

## ✔ 상향식  
- WBS 기반  
- 세부 작업 단위별 비용 합산

## ✔ COCOMO  
- 개발 규모(LOC) 기반  
- 조직형 / 반분리형 / 임베디드형

---

# 📌 **3. 요구사항 분석 – 핵심 키워드**
- 기능 요구사항 vs 비기능 요구사항  
- 요구사항 검증: 완전성, 일관성, 검증 가능성  
- 요구사항 분석 기법: UML, Use Case, DFD

---

# 📌 **4. 프로젝트 관리 – PMBOK 핵심**
## ✔ 5대 프로세스  
- **착수 → 계획 → 실행 → 감시통제 → 종료**

## ✔ 10대 지식 영역  
- 범위, 일정, 비용, 품질, 자원, 의사소통, 위험, 조달, 이해관계자, 통합관리

## ✔ 간트차트 / PERT / CPM  
- PERT: 낙관·비관·기대치  
- CPM: 임계경로(여유시간 0)

---

# 📌 **5. 형상관리(Configuration Management)**
## ✔ 형상관리 4대 요소  
- **형상 식별 → 형상 통제 → 형상 감사 → 형상 기록/보고**

## ✔ 버전 관리 도구  
- Git, SVN  
- 체크인/체크아웃

---

# 📌 **6. 소프트웨어 품질 – ISO/IEC 9126**
6가지 품질 특성 암기  
- **기능성**  
- **신뢰성**  
- **사용성**  
- **효율성**  
- **유지보수성**  
- **이식성**

---

# 📌 **7. 소프트웨어 테스트 – 단골 출제**
## ✔ 테스트 레벨  
- 단위 → 통합 → 시스템 → 인수

## ✔ 테스트 기법  
- 화이트박스: 구문/조건/경로  
- 블랙박스: 동등분할, 경계값, 원인-결과 그래프

## ✔ 통합 테스트  
- 상향식: 드라이버  
- 하향식: 스텁  
- 빅뱅식

---

# 📌 **8. DB 구축 관리 – 핵심만**
- 정규화: 1NF → 2NF → 3NF → BCNF  
- 무결성: 개체, 참조, 도메인  
- 트랜잭션: ACID(원자성, 일관성, 고립성, 지속성)

---

# 📌 **9. 네트워크/보안 – 자주 나오는 부분**
- OSI 7계층  
- 방화벽, IDS, IPS  
- VPN, DMZ  
- 대칭키 vs 공개키  
- 백업: 전체/증분/차등

---

# 📌 **10. IT 서비스 관리 – ITIL 핵심**
- 서비스 전략  
- 서비스 설계  
- 서비스 전환  
- 서비스 운영  
- 지속적 서비스 개선

---

# 🎯 **시험 직전 30초 암기 버전**
- 개발방법론: 폭포수/프로토타입/나선형/애자일  
- 비용산정: 하향식(전문가), 상향식(WBS), COCOMO  
- PMBOK: 착·계·실·감·종  
- 테스트: 단위→통합→시스템→인수  
- 형상관리: 식별→통제→감사→기록  
- 품질: 기능·신뢰·사용·효율·유지·이식  
- DB: 정규화, 무결성, ACID  
- 보안: 방화벽/IDS/IPS, 대칭·공개키  
- ITIL: 전략→설계→전환→운영→개선
