# 모델링/모델화
---

## 1️⃣ 모델링(모델화) 한 줄 정의 (IT 관점)

> **모델링(Modeling)**이란
> 현실 세계의 복잡한 대상이나 문제를
> **목적에 맞게 단순화하여 표현하는 것**이다.

---

## 2️⃣ 지도 🗺️ → 데이터/시스템 모델링

### 📌 일상 모델

* 실제 도시는 건물, 사람, 도로로 매우 복잡
* 지도는 **도로, 주요 건물만 단순화**

### 📌 IT 모델링과 연결

* 실제 업무 프로세스 → **ERD, UML**
* 모든 정보를 표현하지 않고 **필요한 요소만 표현**

### 🔑 핵심

> “현실 그대로가 아니라, **목적에 맞게 생략**한다”

---

## 3️⃣ 건축 설계도 🏠 → 시스템 모델링

### 📌 일상 모델

* 실제 집을 먼저 짓지 않음
* 설계도로 구조, 배선, 배관 확인

### 📌 IT 모델링과 연결

* 시스템 구축 전 **아키텍처 다이어그램**
* 클래스 다이어그램, 컴포넌트 다이어그램

### 🔑 핵심

> “만들기 전에 **미리 그려보는 것**”

---

## 4️⃣ 메뉴판 🍽️ → 기능 모델링

### 📌 일상 모델

* 주방 전체 과정을 보여주지 않음
* 고객에게 필요한 정보만 제공

### 📌 IT 모델링과 연결

* 사용자 관점의 **Use Case Diagram**
* 시스템 내부 로직은 숨김

### 🔑 핵심

> “사용자가 보는 **기능 중심 모델**”

---

## 5️⃣ 가계부 📒 → 데이터 모델링

### 📌 일상 모델

* 모든 소비 기록 X
* 날짜, 금액, 항목만 정리

### 📌 IT 모델링과 연결

* 현실 데이터 → **테이블, 컬럼**
* ERD, 정규화

### 🔑 핵심

> “정보를 **구조화**해서 관리”

---

## 6️⃣ 일정표 📅 → 프로세스 모델링

### 📌 일상 모델

* 하루 전체를 시간 단위로 나눔
* 해야 할 일의 흐름 표현

### 📌 IT 모델링과 연결

* 업무 흐름 → **순서도, 활동 다이어그램**
* 프로세스 중심 모델링

### 🔑 핵심

> “일의 **흐름과 순서**를 표현”

---

## 7️⃣ 축소 모형 🚗 → 시뮬레이션 모델링

### 📌 일상 모델

* 실제 크기 대신 축소된 모형
* 기능·구조 검증

### 📌 IT 모델링과 연결

* 성능 테스트
* 시뮬레이션 모델

### 🔑 핵심

> “작게 만들어 **미리 검증**”

---

## 8️⃣ 교통 신호 체계 🚦 → 규칙/논리 모델링

### 📌 일상 모델

* 복잡한 교통 상황을 신호로 제어
* 규칙 기반 동작

### 📌 IT 모델링과 연결

* 조건, 상태 모델
* 상태 다이어그램

### 🔑 핵심

> “조건에 따른 **행동 규칙 표현**”

---

## 9️⃣ 레시피 📄 → 알고리즘 모델링

### 📌 일상 모델

* 요리 순서와 방법 정리
* 누구나 따라 할 수 있음

### 📌 IT 모델링과 연결

* 문제 해결 절차
* 알고리즘, 의사코드

### 🔑 핵심

> “행동을 **단계화**”

---

## 🔟 시험/강의용 한 문장 예시 ✍️

> 모델링이란 현실 세계의 복잡한 대상이나 문제를 목적에 맞게 단순화하여 표현하는 것이며, 지도나 설계도처럼 전체를 그대로 옮기는 것이 아니라 필요한 요소만 추상화하여 표현하는 것이다.

---

## 🧠 모델링 핵심 키워드 요약

* 추상화 (Abstraction)
* 단순화 (Simplification)
* 목적 중심
* 구조화
* 시각화
* 의사소통 도구

---

## 한 줄로 끝내기 🔥

> **모델링은 “현실을 그대로 옮기는 것”이 아니라
> “이해하고 만들기 쉽게 줄여 그리는 것”이다.**
