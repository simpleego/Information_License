#  프로그래밍 언어 활용 핵심 내용
---

# 📌 1. **프로그래밍 패러다임(Paradigm) – 무조건 나오는 부분**
### ✔ 절차지향(Procedural)
- 순차적 처리  
- 대표 언어: C  
- 장점: 실행 속도 빠름  
- 단점: 유지보수 어려움

### ✔ 객체지향(OOP)
- 캡슐화, 상속, 다형성, 추상화  
- 대표 언어: Java, C++, Python  
- 장점: 재사용성↑, 유지보수↑

### ✔ 함수형(Functional)
- 순수 함수, 불변성  
- 대표 언어: LISP, Haskell  
- 특징: 상태 변화 없음

### ✔ 선언형(Declarative)
- “무엇을 할지”만 기술  
- SQL, HTML

---

# 📌 2. **객체지향 핵심 개념 – 매년 출제**
### ✔ 캡슐화  
데이터 + 메서드를 하나로 묶고 외부 접근 제한

### ✔ 상속  
기존 클래스의 속성과 기능을 물려받음

### ✔ 다형성  
같은 이름의 메서드가 상황에 따라 다르게 동작  
- 오버로딩: 같은 이름, 매개변수 다름  
- 오버라이딩: 상속받은 메서드를 재정의

### ✔ 추상화  
공통 특징만 뽑아 모델링

---

# 📌 3. **자료형(Data Type) – 기본형 vs 참조형**
### ✔ 기본형(Primitive)
정수, 실수, 문자, 논리 등  
메모리에 값 자체 저장

### ✔ 참조형(Reference)
배열, 클래스, 인터페이스  
메모리 주소 저장

---

# 📌 4. **자료구조 – 시험에 자주 나오는 특징**
| 자료구조 | 특징 | 예 |
|---------|------|----|
| 스택 | LIFO | 함수 호출 |
| 큐 | FIFO | 프린터 대기열 |
| 트리 | 계층 구조 | 파일 시스템 |
| 그래프 | 노드 + 간선 | SNS 관계 |
| 해시 | 키-값 매핑 | 사전 구조 |

---

# 📌 5. **정렬 알고리즘 – 시간복잡도 암기**
| 정렬 | 평균 | 최악 | 특징 |
|------|------|------|------|
| 버블 | O(n²) | O(n²) | 인접 비교 |
| 선택 | O(n²) | O(n²) | 최소값 선택 |
| 삽입 | O(n²) | O(n²) | 거의 정렬된 경우 빠름 |
| 퀵 | O(n log n) | O(n²) | 분할 정복 |
| 병합 | O(n log n) | O(n log n) | 안정적, 추가 메모리 필요 |

퀵 vs 병합 구분 문제 자주 나옴.

---

# 📌 6. **제어문 – 기본 문법 체크**
- if / switch  
- for / while / do-while  
- break, continue  
- 예외 처리 try–catch–finally

특히 **switch는 정수/문자 기반**,  
**문자열은 언어에 따라 지원 여부 다름** → 자주 출제.

---

# 📌 7. **함수(Function) / 메서드(Method)**
- 매개변수 전달 방식: 값 전달 vs 참조 전달  
- 재귀 함수 특징  
- 지역 변수 vs 전역 변수

---

# 📌 8. **메모리 구조 – 시험 단골**
| 영역 | 특징 |
|------|------|
| 코드(Code) | 프로그램 명령어 |
| 데이터(Data) | 전역/정적 변수 |
| 스택(Stack) | 지역 변수, 함수 호출 |
| 힙(Heap) | 동적 메모리 |

스택 오버플로우, 힙 메모리 누수 관련 문제 자주 나옴.

# 📌 10. **라이브러리 / API / 프레임워크**
- 라이브러리: 개발자가 호출  
- 프레임워크: 프레임워크가 개발자 코드를 호출(Inversion of Control)  
- API: 기능을 제공하는 인터페이스
