# 정보시스템 구축관리

## 1. 소프트웨어 개발 방법론 및 생명주기 (SDLC)

가장 기초가 되며 매회 출제되는 영역입니다.

* **폭포수 모델(Waterfall):** 순차적인 실행, 가장 오래된 모델, 피드백이 어려움.
* **나선형 모델(Spiral):** **'위험 분석'**이 핵심 키워드 (계획 -> 위험 분석 -> 개발 -> 평가 반복).
* **애자일(Agile):** 고객과의 소통 중시, 짧은 개발 주기. (XP, Scrum, Kanban 등)
* **XP(eXtreme Programming) 5대 가치:** 용기, 단순성, 커뮤니케이션, 피드백, 존중.

## 2. 암호화 알고리즘 (보안 핵심)

암호화 방식의 차이점을 묻는 문제가 단골로 등장합니다.

| 구분 | 양방향 - 대칭키 (개인키) | 양방향 - 비대칭키 (공개키) | 단방향 |
| --- | --- | --- | --- |
| **특징** | 암호화/복호화 키가 동일, 속도 빠름 | 키가 서로 다름, 관리 용이, 속도 느림 | 복호화 불가 (Hash) |
| **종류** | **블록:** DES, AES, SEED, ARIA<br>

<br>**스트림:** RC4 | RSA (소인수분해), ElGamal | SHA-series, MD5 |

## 3. 서비스 공격 유형 및 보안 솔루션

최근 보안 비중이 높아지면서 반드시 숙지해야 합니다.

* **DoS / DDoS 공격:**
* **SYN Flooding:** 3-way handshaking의 취약점 이용.
* **Smurfing:** ICMP 패킷과 위조된 IP 주소 이용.
* **공격 도구:** Trin00, TFN, Stacheldraht.


* **기타 공격:** * **Phishing:** 가짜 사이트 유도.
* **Sniffing:** 네트워크 패킷 도청.
* **Spoofing:** 승인받은 사용자인 것처럼 속임 (IP, ARP 등).


* **보안 솔루션:**
* **IDS(침입 탐지 시스템):** 탐지 중심.
* **IPS(침입 방지 시스템):** 탐지 후 차단까지 수행.
* **SIEM:** 다양한 보안 장비의 로그를 통합 관리.

## 4. 네트워크 및 신기술 용어

기술 명칭과 정의를 연결하는 문제가 많습니다.

* **RAID:** 여러 개의 하드디스크를 하나로 묶어 성능과 안정성을 높이는 기술. (RAID 0: 스트라이핑, RAID 1: 미러링)
* **Infrastructure:**
* **PaaS:** 플랫폼 제공 (개발 환경).
* **IaaS:** 인프라 제공 (서버, 스토리지).
* **SaaS:** 소프트웨어 제공 (응용 프로그램).


* **네트워크 장비:**
* **Gateway:** 서로 다른 프로토콜 네트워크 연결.
* **Router:** 최적의 경로(Path) 설정.

## 5. 비용 산정 모델

계산식보다는 각 모델의 특징을 묻는 경우가 많습니다.

* **COCOMO:** 보헴(Boehm)이 제안, 프로그램 규모(LOC)에 따라 비용 산정.
* **Organic(5만 라인 이하), Semi-Detached(30만 이하), Embedded(30만 이상).**


* **Putnam:** 노력 분포도를 이용, **Rayleigh-Norden 곡선**을 활용.
* **Function Point (FP):** 기능 점수 모형, 사용자 관점의 기능을 수치화.

---

### 💡 마지막 암기 팁

1. **세션 하이재킹(Session Hijacking):** 연결된 세션을 가로채는 공격.
2. **형상 관리(SCM):** 소프트웨어 변경 사항을 관리하는 것 (Git, SVN 등).
3. **결합도(Coupling)는 낮추고, 응집도(Cohesion)는 높여야** 좋은 설계입니다.
