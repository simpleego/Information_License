#   소프트웨어 설계_개발_구현
> **“윈도우 기반 간단한 사칙연산 계산기”**를 예로 들어, **소프트웨어 설계 → 개발 → 구현** 단계별로 어떤 일을 해야 하는지 정리

---

## 🏗 1단계: 소프트웨어 설계 (설계도 그리기 단계)
- **요구사항 정의**  
  - 계산기는 덧셈, 뺄셈, 곱셈, 나눗셈을 지원해야 한다.  
  - 윈도우 환경에서 실행되는 GUI 프로그램이어야 한다.  
  - 사용자가 숫자를 입력하고 버튼을 눌러 결과를 확인할 수 있어야 한다.
- **UI 설계**  
  - 입력창(TextBox) 1개  
  - 숫자 버튼(0~9)  
  - 연산 버튼(+, -, ×, ÷)  
  - 결과 버튼(=)과 초기화 버튼(C)  
- **기능 설계**  
  - 입력값을 받아 연산 수행  
  - 연산 결과를 화면에 출력  
  - 오류 처리(예: 0으로 나누기 시 에러 메시지 표시)
- **모듈 설계**  
  - **UI 모듈**: 버튼, 입력창, 결과창  
  - **연산 모듈**: 덧셈, 뺄셈, 곱셈, 나눗셈 함수  
  - **제어 모듈**: 버튼 이벤트 처리, 연산 호출, 결과 전달  

👉 이 단계는 **집을 짓기 전에 설계도 그리는 과정**과 같아요.

---

## 💻 2단계: 소프트웨어 개발 (코드 작성 단계)
- **프로그래밍 언어 선택**: 예를 들어 C# (WinForms/WPF), Java (Swing/JavaFX), Python (Tkinter/PyQt)  
- **UI 코드 작성**  
  - 버튼과 입력창을 화면에 배치  
  - 각 버튼에 이벤트 핸들러 연결 (예: “+” 버튼 클릭 시 Add 함수 호출)
- **연산 함수 구현**  
  ```csharp
  int Add(int a, int b) { return a + b; }
  int Sub(int a, int b) { return a - b; }
  int Mul(int a, int b) { return a * b; }
  double Div(int a, int b) {
      if(b == 0) throw new DivideByZeroException();
      return (double)a / b;
  }
  ```
- **테스트 코드 작성**  
  - 2+3=5, 10-4=6, 6×7=42, 8÷2=4 같은 기본 케이스 확인  
  - 0으로 나누는 경우 예외 처리 확인  

👉 이 단계는 **설계도를 바탕으로 실제 집을 짓는 과정**이에요.

---

## 🚀 3단계: 소프트웨어 구현 (배포·실행 단계)
- **빌드(Build)**  
  - 작성한 코드를 컴파일해서 실행 파일(.exe) 생성  
- **테스트(Test)**  
  - 실제 윈도우 환경에서 실행해 UI 동작과 연산 결과 확인  
- **배포(Deployment)**  
  - 설치 파일(.msi 등)로 패키징해서 사용자 PC에 설치 가능하게 제공  
- **유지보수(Maintenance)**  
  - 버그 수정, 기능 추가(예: 제곱, 루트 연산)  

👉 이 단계는 **완성된 집을 사람들이 실제로 들어가서 쓰도록 열어주는 과정**이에요.
