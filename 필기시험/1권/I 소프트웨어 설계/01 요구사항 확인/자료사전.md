# 자료사전
> 자료사전(Data Dictionary)은 데이터베이스나 시스템에서 사용하는 **데이터 항목들의 정의, 속성, 관계** 등을 정리한 문서

---

# 📘 자료사전 예시

| 데이터 항목 | 설명 | 데이터 타입 | 길이 | 값의 범위/도메인 | 예시 값 |
|-------------|------|-------------|------|------------------|---------|
| 고객ID | 고객을 구분하는 고유 번호 | 정수(Integer) | 10자리 | 1 이상 | 1000123456 |
| 이름 | 고객의 성명 | 문자열(String) | 50자 | 한글/영문 | 김철수 |
| 생년월일 | 고객의 출생일 | 날짜(Date) | YYYY-MM-DD | 1900-01-01 ~ 현재 | 1990-05-21 |
| 이메일 | 고객의 이메일 주소 | 문자열(String) | 100자 | 이메일 형식 | test@example.com |
| 전화번호 | 고객의 연락처 | 문자열(String) | 15자 | 숫자/하이픈 포함 | 010-1234-5678 |
| 가입일자 | 시스템 가입일 | 날짜(Date) | YYYY-MM-DD | 현재 날짜 기준 | 2026-01-10 |

---

이런 식으로 **각 데이터 항목의 정의, 타입, 제약조건, 예시 값**을 정리
실제 프로젝트에서는 더 많은 속성(예: 기본키 여부, 외래키 관계, Null 허용 여부 등)을 포함

# 자료사전 기호를 이용한 표현
알겠어. 자료사전에서는 흔히 **자료사전 기호**(Data Dictionary Symbols)를 사용해서 데이터 흐름도와 연결해. 기본적으로 다음과 같은 기호를 씀:

- `=` : 정의(데이터 항목을 구성하는 요소 설명)  
- `+` : 합성(여러 데이터 항목을 합쳐서 구성)  
- `{ }` : 반복(여러 개 존재 가능)  
- `[ ]` : 선택(있을 수도 있고 없을 수도 있음)  

위의 표 내용을 자료사전 기호로 간단히 표현하면 이렇게 정리할 수 있어:

---

# 📘 자료사전 기호 활용 예시

```
고객정보 = 고객ID + 이름 + 생년월일 + 이메일 + 전화번호 + 가입일자

고객ID = 정수(10자리)
이름 = 문자열(50자)
생년월일 = 날짜(YYYY-MM-DD)
이메일 = 문자열(100자)
전화번호 = 문자열(15자)
가입일자 = 날짜(YYYY-MM-DD)

고객목록 = {고객정보}
```

---

이렇게 하면 **고객정보**라는 데이터 항목이 어떤 세부 요소들로 구성되는지, 그리고 반복 구조(목록 형태)까지 표현할 수 있어.  

👉 필요하다면 `[]` 기호를 써서 선택적 항목(예: 이메일은 필수, 전화번호는 선택)도 표시할 수 있음:

```
고객정보 = 고객ID + 이름 + 생년월일 + 이메일 + [전화번호] + 가입일자
```
