# UML 예시
---

# 🧱 1) **클래스 다이어그램 (Class Diagram)**  
객체지향 설계에서 가장 많이 쓰는 UML.  
클래스의 **속성, 메서드, 관계(상속/연관/의존)**를 표현해.

### 📘 예제: 회원과 주문 시스템

```
+------------------+
|     Member       |
+------------------+
| - id: int        |
| - name: String   |
| - email: String  |
+------------------+
| + login()        |
| + logout()       |
+------------------+

          1    *
Member ----------------> Order
             (has)
+------------------+
|      Order       |
+------------------+
| - orderId: int   |
| - date: Date     |
| - amount: int    |
+------------------+
| + cancel()       |
+------------------+
```

👉 **설명**  
- Member 1명이 Order 여러 개를 가질 수 있는 **1:N 연관 관계**  
- 클래스의 속성과 메서드가 명확히 보임  

---

# 🧱 2) **객체 다이어그램 (Object Diagram)**  
클래스 다이어그램의 **실제 인스턴스 상태**를 보여줌.

### 📘 예제: 실제 객체 상태

```
member1: Member
-------------------------
id = 101
name = "Kim"
email = "kim@test.com"

order1: Order
-------------------------
orderId = 5001
date = "2024-01-01"
amount = 30000
```

👉 **설명**  
- 특정 시점에 실제 객체가 어떤 값을 가지고 있는지 표현  

---

# 🧱 3) **유스케이스 다이어그램 (Use Case Diagram)**  
사용자(Actor)와 시스템의 상호작용을 표현.  
요구사항 분석 단계에서 사용.

### 📘 예제: 로그인 기능

```
     +-------------+
     |   사용자    |
     +-------------+
           |
           | uses
           v
   +-------------------+
   |   로그인 하기     |
   +-------------------+
           |
           | includes
           v
   +-------------------+
   |  비밀번호 확인    |
   +-------------------+
```

👉 **설명**  
- Actor: 사용자  
- Use Case: 로그인 하기  
- includes: 로그인 과정에서 비밀번호 확인이 포함됨  

---

# 🧱 4) **시퀀스 다이어그램 (Sequence Diagram)**  
객체 간 메시지 흐름을 **시간 순서대로** 표현.

### 📘 예제: 로그인 처리 흐름

```
User        UI        AuthService       DB
 |          |             |             |
 |---clickLogin()-------->|             |
 |                        |---verify()-->| 
 |                        |              |---query()-->|
 |                        |              |<--result-----|
 |                        |<--true-------|
 |<--loginSuccess()-------|
```

👉 **설명**  
- 시간 흐름은 위에서 아래로  
- 객체 간 메시지 전달이 명확하게 보임  

---

# 🧱 5) **활동 다이어그램 (Activity Diagram)**  
업무 흐름도와 비슷.  
조건, 분기, 반복 등 **프로세스 흐름**을 표현.

### 📘 예제: 주문 처리 흐름

```
   [시작]
      |
      v
  주문 접수
      |
      v
재고 확인 -----> [재고 없음] -----> 주문 취소
      |
[재고 있음]
      |
      v
결제 처리
      |
      v
  주문 완료
      |
     [끝]
```

👉 **설명**  
- 조건 분기(재고 있음/없음) 표현  
- 전체 프로세스 흐름을 한눈에 파악 가능  

---

# 🧱 6) **상태 다이어그램 (State Machine Diagram)**  
객체의 **상태 변화**를 표현.

### 📘 예제: 주문 상태

```
 [주문 생성]
       |
       v
   결제 대기
       |
       v
   결제 완료
       |
       v
   배송 중
       |
       v
   배송 완료
```

👉 **설명**  
- 주문이 어떤 상태를 거쳐 최종 완료되는지 표현  

---

# 🧱 7) **컴포넌트 다이어그램 (Component Diagram)**  
시스템을 구성하는 **컴포넌트(모듈)** 간 관계 표현.

### 📘 예제

```
+------------------+       +------------------+
|  Web Controller  | ----> |  Auth Service    |
+------------------+       +------------------+
                                  |
                                  v
                         +------------------+
                         |   User DB        |
                         +------------------+
```

👉 **설명**  
- 웹 컨트롤러 → 인증 서비스 → DB  
- 시스템 구조를 상위 레벨에서 표현  

---

# 🧱 8) **배치 다이어그램 (Deployment Diagram)**  
서버, 장비, 네트워크 등 **물리적 배치**를 표현.

### 📘 예제

```
+------------------------+
|   Web Server           |
|  (Nginx + Spring)      |
+------------------------+
            |
            | HTTP
            v
+------------------------+
|   DB Server            |
|   (MySQL)              |
+------------------------+
```

👉 **설명**  
- 실제 운영 환경에서 어떤 서버가 어디에 배치되는지 표현  

---

# 🧱 9) **커뮤니케이션 다이어그램 (Communication Diagram)**  
시퀀스 다이어그램과 비슷하지만  
**객체 간 연결 관계**에 초점.

### 📘 예제

```
User <----> UI <----> AuthService <----> DB
   1: clickLogin
   2: verify
   3: query
   4: result
```

---

# 🧱 10) **타이밍 다이어그램 (Timing Diagram)**  
시간에 따른 상태 변화를 표현.  
임베디드/하드웨어에서 많이 사용.

### 📘 예제

```
시간 → → → →

상태:  Idle ---- Running ---- Idle ---- Error
```
