# 데이터 포맷
- JSON
- XML
- YAML
---

# 🟦 1) JSON (JavaScript Object Notation)

📌 **가장 많이 쓰이는 데이터 형식**  
- 웹/앱 개발에서 표준처럼 사용  
- 가볍고, 사람이 읽기 쉬움  
- 구조: `{ }`, `[ ]`, `"key": "value"`

### ✔ 예시  
학생 정보를 JSON으로 표현하면:

```json
{
  "name": "Jong",
  "age": 25,
  "skills": ["Java", "Python", "SQL"]
}
```

### ✔ 특징  
- 깔끔하고 간단  
- 웹 API(REST)에서 기본 형식  
- 데이터 교환에 최적화  

---

# 🟧 2) XML (eXtensible Markup Language)

📌 **태그 기반 구조**, HTML과 비슷  
- 데이터뿐 아니라 **데이터의 의미(메타데이터)**도 함께 표현  
- SOAP, 금융/공공기관 시스템에서 많이 사용

### ✔ 예시  
같은 데이터를 XML로 표현하면:

```xml
<Student>
  <Name>Jong</Name>
  <Age>25</Age>
  <Skills>
    <Skill>Java</Skill>
    <Skill>Python</Skill>
    <Skill>SQL</Skill>
  </Skills>
</Student>
```

### ✔ 특징  
- 태그가 많아서 **장황하지만 구조가 명확**  
- 문서 형태로 표현하기 좋음  
- 금융/의료/공공기관에서 여전히 많이 사용  

---

# 🟩 3) YAML (YAML Ain’t Markup Language)

📌 **사람이 읽기 가장 쉬운 형식**  
- 들여쓰기 기반  
- 설정 파일(config)에서 많이 사용  
- JSON보다 더 간결

### ✔ 예시  
같은 데이터를 YAML로 표현하면:

```yaml
name: Jong
age: 25
skills:
  - Java
  - Python
  - SQL
```

### ✔ 특징  
- 들여쓰기만으로 구조 표현  
- 매우 간결하고 읽기 쉬움  
- DevOps, Docker, Kubernetes 설정 파일에서 필수  

---

# 🎯 jong을 위한 핵심 비교 요약

| 형식 | 장점 | 단점 | 주 사용처 |
|------|------|------|-----------|
| **JSON** | 가볍고 간단, 웹 표준 | 주석 불가 | 웹/앱 API |
| **XML** | 구조·메타데이터 표현 강력 | 태그가 많아 복잡 | 금융, 공공, SOAP |
| **YAML** | 사람이 읽기 가장 쉬움 | 들여쓰기 오류 위험 | DevOps, 설정 파일 |

---

# 🧠 한 줄로 정리하면  
- **JSON**: 웹에서 가장 많이 쓰는 가볍고 간단한 데이터 형식  
- **XML**: 태그 기반, 문서 구조 표현에 강함  
- **YAML**: 사람이 읽기 쉬운 설정 파일용 형식
