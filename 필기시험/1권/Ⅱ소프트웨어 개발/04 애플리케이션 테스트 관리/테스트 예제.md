# 🧪 1) **API 테스트 예 (REST API)**

## 테스트 이름  
**회원 로그인 API 정상 응답 확인**

## 목적  
올바른 계정 정보로 로그인 요청 시 정상적으로 토큰이 발급되는지 확인한다.

## 요청  
- **POST** `/api/v1/login`  
- Body:
```json
{
  "username": "test01",
  "password": "1234"
}
```

## 기대 결과  
- HTTP Status: **200 OK**  
- Response Body:
```json
{
  "token": "<JWT_TOKEN>",
  "expires_in": 3600
}
```

# 🧪 2) **시나리오 기반 테스트 예 (UI/기능 흐름)**

## 시나리오  
**상품을 장바구니에 담고 결제 페이지로 이동**

## 절차  
1. 메인 페이지 접속  
2. 상품 목록에서 임의의 상품 선택  
3. "장바구니 담기" 버튼 클릭  
4. 장바구니 페이지 이동  
5. "결제하기" 버튼 클릭  

## 기대 결과  
- 선택한 상품이 장바구니에 표시됨  
- 수량 기본값은 1  
- 결제 페이지로 정상 이동  
- 총 금액이 정확히 계산됨

# 🧪 3) **성능 테스트 예 (JMeter 스타일)**

## 테스트 이름  
**로그인 API 부하 테스트**

## 목적  
1초당 100명의 사용자가 동시에 로그인 요청을 보낼 때 서버가 정상적으로 응답하는지 확인한다.

## 설정  
- Thread Group: **100 users**  
- Ramp-up: **1 second**  
- Loop Count: **10**

## 성공 기준  
- 평균 응답 시간: **300ms 이하**  
- 에러율: **1% 미만**  
- 95th Percentile 응답 시간: **500ms 이하**
