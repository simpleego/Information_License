# **OOSE / OMT / OOD** 비교
---

# 🧩 1) **OOSE (Object-Oriented Software Engineering)**  
*Ivar Jacobson*이 만든 방법론.  
**유스케이스(Use Case)** 개념을 처음 도입한 것으로 유명해.

### ✔ 핵심 특징
- **유스케이스 중심 개발**  
- 사용자 요구사항을 시나리오 형태로 분석  
- 객체 모델링 + 동적 모델링 + 기능 모델링 포함  
- UML의 기반이 된 방법론 중 하나

### ✔ 구성 모델
- **유스케이스 모델**
- **분석 객체 모델**
- **상호작용 모델(Sequence/Collaboration)**
- **상태 모델(State)**

### ✔ 장점
- 사용자 요구사항을 명확히 파악  
- 실제 사용 시나리오 기반이라 실무 친화적

---

# 🧩 2) **OMT (Object Modeling Technique)**  
*James Rumbaugh*가 만든 방법론.  
객체 모델링을 가장 체계적으로 정리한 초기 방법론.

### ✔ 핵심 특징
- **정적 모델링(객체 모델)**에 강점  
- 시스템을 3가지 모델로 나누어 분석

### ✔ 3가지 모델
1. **객체 모델(Object Model)** – 클래스, 속성, 관계  
2. **동적 모델(Dynamic Model)** – 상태도, 이벤트  
3. **기능 모델(Functional Model)** – 데이터 흐름도(DFD)

### ✔ 장점
- 구조적 분석과 객체지향 분석을 연결  
- 복잡한 시스템 구조를 명확하게 표현

---

# 🧩 3) **OOD (Object-Oriented Design)**  
*Grady Booch*가 만든 객체지향 설계 방법론.  
설계 중심이며, 클래스 설계·관계 정의에 강함.

### ✔ 핵심 특징
- **객체지향 설계 원칙 중심**  
- 클래스, 객체, 메시지, 관계를 상세히 설계  
- 복잡한 시스템을 계층적으로 구조화

### ✔ 주요 산출물
- 클래스 다이어그램  
- 객체 다이어그램  
- 모듈 구조  
- 런타임 구조

### ✔ 장점
- 설계 단계에서 매우 체계적  
- 대규모 시스템 설계에 적합

# **COAD–Yourdon**
> OOSE, OMT, OOD와 함께 **초기 객체지향 4대 방법론** 중 하나

---

# 🧩 **COAD–Yourdon 방법론(Coad & Yourdon Method)**  
Peter Coad와 Edward Yourdon이 만든 **객체지향 분석(Object-Oriented Analysis, OOA)** 중심의 방법론이야.

---

# 🧱 1) 핵심 개념  
COAD–Yourdon은 **현실 세계의 객체를 중심으로 시스템을 분석**하는 접근 방식이야.

### 주요 특징
- 객체(Object) 중심 분석  
- 객체 간 **관계(Relationship)** 강조  
- **5단계 분석 절차**로 유명  
- 복잡한 시스템을 객체 단위로 나누어 이해하기 쉽게 만듦  
- UML 이전 시대의 대표적 객체지향 분석 기법

---

# 🧱 2) COAD–Yourdon의 5단계 분석 절차

### ✔ 1) 객체 식별(Object Identification)  
현실 세계에서 시스템을 구성하는 객체를 찾는 단계  
예: 고객, 주문, 상품 등

### ✔ 2) 구조 식별(Structure Identification)  
객체 간의 **상속 관계**, **집합 관계** 등을 정의  
예: 직원 → 정규직/계약직

### ✔ 3) 주체 식별(Subject Identification)  
시스템을 여러 **주제 영역(Subject Area)**으로 나누어 복잡성을 줄임  
예: 주문 관리, 재고 관리, 사용자 관리

### ✔ 4) 속성 및 서비스 정의(Attribute & Service Definition)  
각 객체가 가지는 **속성(데이터)**과 **서비스(메서드)**를 정의  
예:  
- 고객: 이름, 주소, 연락처  
- 서비스: 주문하기(), 정보수정()

### ✔ 5) 객체 간 관계 정의(Relationship Definition)  
객체 간의 연관, 집합, 사용 관계 등을 명확히 표현  
예: 고객 — 주문(1:N)

---

# 🧱 3) COAD–Yourdon의 장점

- 현실 세계 모델링에 강함  
- 객체 중심 사고를 쉽게 도입 가능  
- 복잡한 시스템을 주제 영역으로 나누어 관리  
- UML 등장 이전에 널리 사용되던 체계적 방법론

---

# 🧱 4) 다른 객체지향 방법론과 비교

| 방법론 | 특징 | 중심 |
|--------|--------|--------|
| **OOSE** | 유스케이스 기반 | 요구사항 중심 |
| **OMT** | 객체·동적·기능 모델 3분할 | 구조 중심 |
| **OOD(Booch)** | 설계 중심, 클래스 구조화 | 설계 중심 |
| **COAD–Yourdon** | 객체 식별 + 관계 중심 | 분석 중심 |

👉 COAD–Yourdon은 **분석(Analysis)**에 가장 강한 방법론이야.

**COAD–Yourdon = 객체를 찾고, 구조화하고, 관계를 정의하는 객체지향 분석 방법론**
