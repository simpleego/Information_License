# **OOSE / OMT / OOD** 비교
---

# 🧩 1) **OOSE (Object-Oriented Software Engineering)**  
*Ivar Jacobson*이 만든 방법론.  
**유스케이스(Use Case)** 개념을 처음 도입한 것으로 유명해.

### ✔ 핵심 특징
- **유스케이스 중심 개발**  
- 사용자 요구사항을 시나리오 형태로 분석  
- 객체 모델링 + 동적 모델링 + 기능 모델링 포함  
- UML의 기반이 된 방법론 중 하나

### ✔ 구성 모델
- **유스케이스 모델**
- **분석 객체 모델**
- **상호작용 모델(Sequence/Collaboration)**
- **상태 모델(State)**

### ✔ 장점
- 사용자 요구사항을 명확히 파악  
- 실제 사용 시나리오 기반이라 실무 친화적

---

# 🧩 2) **OMT (Object Modeling Technique)**  
*James Rumbaugh*가 만든 방법론.  
객체 모델링을 가장 체계적으로 정리한 초기 방법론.

### ✔ 핵심 특징
- **정적 모델링(객체 모델)**에 강점  
- 시스템을 3가지 모델로 나누어 분석

### ✔ 3가지 모델
1. **객체 모델(Object Model)** – 클래스, 속성, 관계  
2. **동적 모델(Dynamic Model)** – 상태도, 이벤트  
3. **기능 모델(Functional Model)** – 데이터 흐름도(DFD)

### ✔ 장점
- 구조적 분석과 객체지향 분석을 연결  
- 복잡한 시스템 구조를 명확하게 표현

---

# 🧩 3) **OOD (Object-Oriented Design)**  
*Grady Booch*가 만든 객체지향 설계 방법론.  
설계 중심이며, 클래스 설계·관계 정의에 강함.

### ✔ 핵심 특징
- **객체지향 설계 원칙 중심**  
- 클래스, 객체, 메시지, 관계를 상세히 설계  
- 복잡한 시스템을 계층적으로 구조화

### ✔ 주요 산출물
- 클래스 다이어그램  
- 객체 다이어그램  
- 모듈 구조  
- 런타임 구조

### ✔ 장점
- 설계 단계에서 매우 체계적  
- 대규모 시스템 설계에 적합

# 🧠 **OOSE / OMT / OOD 비교 요약**

| 구분 | OOSE | OMT | OOD |
|------|------|------|------|
| 창시자 | Ivar Jacobson | James Rumbaugh | Grady Booch |
| 중심 개념 | **유스케이스 중심** | **객체 모델 중심** | **설계 중심** |
| 모델 구성 | 유스케이스, 객체, 동적, 상태 | 객체, 동적, 기능 | 클래스/객체 설계 |
| 강점 | 요구사항 분석 | 구조적 모델링 | 설계 구조화 |
| 활용 단계 | 분석 + 설계 | 분석 중심 | 설계 중심 |
| UML 기여도 | 유스케이스 도입 | 객체 모델링 정립 | 설계 표기법 제공 |
