# Activity 다이어그램
> 시스템이나 프로세스의 **동작 흐름을 시각적으로 표현**하는 도구

---

## 🧭 Activity 다이어그램이란?

### ✅ 정의
- **Activity Diagram**은 UML(Unified Modeling Language)의 한 종류로, **작업의 흐름(프로세스)**을 표현합니다.
- 시스템이 **어떤 순서로 동작하는지**, **조건 분기**, **반복**, **병렬 처리** 등을 시각적으로 보여줍니다.

---

## 🧩 주요 구성 요소

| 구성 요소 | 설명 | 기호 |
|------------|-----------------------------|--------|
| **시작 노드** | 흐름의 시작점 | ● (검은 원) |
| **활동(Activity)** | 수행되는 작업 또는 행동 | ▭ (둥근 사각형) |
| **흐름(Flow)** | 활동 간의 연결 | → (화살표) |
| **결정 노드(Decision)** | 조건 분기 (if/else) | ◆ (마름모) |
| **병합 노드(Merge)** | 여러 흐름을 하나로 합침 | ◆ (마름모) |
| **종료 노드** | 흐름의 종료점 | ◎ (이중 원) |
| **병렬 분기(Fork)** | 동시에 여러 작업 시작 | ──┬── |
| **병렬 병합(Join)** | 병렬 작업이 끝나고 합침 | ──┴── |

---

## 🎯 Activity 다이어그램의 목적

- **비즈니스 프로세스 모델링**: 업무 흐름을 시각적으로 표현
- **Use Case 내부 흐름 설명**: 사용 사례의 세부 동작을 명확히 보여줌
- **조건/반복/병렬 처리 표현**: 복잡한 로직을 쉽게 이해 가능
- **개발자와 사용자 간 커뮤니케이션 도구**로 활용

---

## 🛒 예시: 온라인 쇼핑 흐름

1. 사용자가 상품을 탐색한다  
2. 상품을 선택하고 장바구니에 담는다  
3. 결제 여부를 결정한다  
4. 결제 진행 → 배송 정보 입력 → 결제 성공 여부 판단  
5. 성공 시 주문 확인, 실패 시 오류 처리  

👉 이 흐름을 Activity 다이어그램으로 표현하면, 각 단계가 **활동 노드**로, 조건은 **결정 노드**로, 흐름은 **화살표**로 연결됩니다.
