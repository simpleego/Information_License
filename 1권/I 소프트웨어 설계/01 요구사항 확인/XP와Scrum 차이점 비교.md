# XP와 Scrum 차이점 비교
> **Scrum은 프로젝트 관리 중심의 애자일 프레임워크이고, XP(eXtreme Programming)는 엔지니어링 실천 중심의 애자일 방법론입니다.
> Scrum은 역할과 프로세스에 초점을 맞추고, XP는 코드 품질과 개발 기법에 집중합니다.**

---

## 📝 Scrum vs XP 비교 정리

| 구분 | **Scrum** | **XP (Extreme Programming)** |
|------|-----------|-----------------------------|
| **핵심 초점** | 프로젝트 관리와 팀 협업 | 코드 품질과 엔지니어링 실천 |
| **주기** | 스프린트 (2~4주) | Iteration (1~2주) |
| **역할(Role)** | 제품 책임자(Product Owner), 스크럼 마스터, 개발팀 | 특별한 역할 구분 없음, 개발팀 중심 |
| **주요 활동** | - 스프린트 계획 회의<br>- 데일리 스크럼<br>- 스프린트 리뷰 & 회고 | - 짝 프로그래밍(Pair Programming)<br>- 테스트 주도 개발(TDD)<br>- 지속적 통합(CI)<br>- 리팩토링 |
| **강점** | - 명확한 역할과 책임<br>- 진행 상황 투명성 확보<br>- 고객 피드백 반영 용이 | - 코드 품질 향상<br>- 팀 내 지식 공유 강화<br>- 빠른 요구사항 대응 |
| **적합한 환경** | - 중대형 프로젝트<br>- 명확한 목표와 관리가 필요한 팀 | - 소규모 팀<br>- 기술적 복잡성이 높은 프로젝트 |
| **단점/리스크** | - 팀 자율성과 책임감 부족 시 실패 가능<br>- 일정 관리가 까다로움 | - 엔지니어링 실천법을 꾸준히 유지하기 어려움<br>- 숙련된 개발자가 필요 |

## 📌 핵심 차이 요약
- **Scrum**은 *“누가 무엇을 언제 할지”*를 관리하는 프레임워크로, **역할과 프로세스**가 뚜렷합니다.  
- **XP**는 *“어떻게 개발할지”*에 집중하며, **코드 품질과 개발 기법**을 극단적으로 실천합니다.  
- 두 방법론은 **애자일 선언문**을 기반으로 하지만, Scrum은 관리적 측면, XP는 기술적 측면을 강화합니다.  

---

👉 정리하자면, **Scrum은 프로젝트 관리에 강점**, **XP는 엔지니어링 실천에 강점**이 있습니다. 실제로는 두 방법론을 **함께 적용**하는 경우가 많습니다.
예를 들어 Scrum으로 프로젝트를 관리하면서, XP의 TDD와 페어 프로그래밍을 활용해 코드 품질을 높이는 식이죠.
