# XP / Lean / Scrum

---

## 🔍 XP · Lean · Scrum 비교 표

| 구분      | **XP (eXtreme Programming)** | **Lean**            | **Scrum**         |
| ------- | ---------------------------- | ------------------- | ----------------- |
| 기본 개념   | 개발 기법 중심 애자일 방법론             | 낭비 제거 중심 사고방식       | 팀 운영·관리 중심 프레임워크  |
| 핵심 목표   | **코드 품질 & 변화 대응**            | **가치 극대화 & 낭비 최소화** | **빠른 제품 완성 & 협업** |
| 초점      | 개발자 실천 기법                    | 프로세스 효율             | 팀 역할·이벤트          |
| 반복 주기   | 매우 짧음 (1~2주)                 | 고정 주기 없음            | 스프린트 (1~4주)       |
| 요구사항 변경 | 적극 수용                        | 필요한 것만 유지           | 스프린트 중 변경 제한      |
| 문서      | 최소화                          | 최소화                 | 필요한 만큼            |
| 대표 기법   | TDD, 페어 프로그래밍, 리팩토링          | 낭비 제거, 지속적 개선       | 데일리 스크럼, 스프린트     |

---

## 🧩 핵심 특징 한 방에 정리

### ⚡ XP (eXtreme Programming)

* **개발 실천 방법에 가장 집중**
* 코드 품질 최우선
* 변화 많은 프로젝트에 강함

📌 대표 키워드

> TDD / 페어 프로그래밍 / 지속적 통합 / 리팩토링

---

### 🧘 Lean (린)

* 제조업(도요타)에서 시작
* **쓸데없는 것 다 없애자** 마인드
* 애자일의 철학적 뿌리 느낌

📌 대표 키워드

> 낭비 제거 / 가치 중심 / 지속적 개선 / 최소 기능

---

### 🏃 Scrum

* **가장 대중적인 애자일 프레임워크**
* 역할·회의·산출물이 명확
* 팀 관리와 일정 운영에 강함

📌 대표 키워드

> Product Owner / Scrum Master / 스프린트 / 데일리 스크럼

---

## 🧠 시험 대비 초압축 요약

* **XP** → *개발 기법* 중심
* **Lean** → *철학·사고방식* 중심
* **Scrum** → *팀 운영·관리* 중심

---

## 📝 정보처리기사 단골 서술형 문장

> XP는 개발 실천 기법에 중점을 둔 애자일 방법론이며,
> Lean은 낭비 제거를 통해 가치를 극대화하는 사고방식이고,
> Scrum은 역할과 반복 주기를 명확히 정의한 애자일 프레임워크이다.
