# 칸반보드
> 👉 **작업 흐름을 시각화하여 현재 상태와 병목을 한눈에 파악하는 작업 관리 도구**
> 즉,  “지금 일이 어디까지 왔는지 보드 한 장으로 보여주는 도구”

---

## 📋 기본 구성

| 열(Column)   | 의미      |
| ----------- | ------- |
| To Do       | 해야 할 작업 |
| In Progress | 진행 중    |
| Done        | 완료      |

➡️ 실제 현장에서는 더 세분화됨
(예: 분석 → 설계 → 개발 → 테스트 → 완료)

## 🏷️ 카드(Card)

* 하나의 카드 = 하나의 작업
* 카드에는 보통:

  * 작업명
  * 담당자
  * 우선순위
  * 예상/실제 소요 시간

---

## 🚦 핵심 원칙

### 1️⃣ 시각화(Visualize Work)

* 모든 작업을 보드에 표시
* 숨겨진 일 없음

### 2️⃣ WIP 제한 (Work In Progress)

* 진행 중 작업 수 제한
* 멀티태스킹 방지 → 품질 향상

### 3️⃣ 흐름 관리 (Flow)

* 작업이 **막히는 지점(병목)** 파악
* 지속적 개선

---

## 🔄 칸반보드 vs 스크럼 (한눈 비교)

| 구분    | 칸반         | 스크럼       |
| ----- | ---------- | --------- |
| 반복 주기 | 없음 (연속 흐름) | 스프린트      |
| 역할    | 정해진 역할 없음  | PO, SM, 팀 |
| 변경    | 언제든 가능     | 스프린트 중 제한 |
| 초점    | 작업 흐름      | 일정 관리     |

---

## 🧠 시험용 핵심 문장

> **칸반보드는 작업의 흐름을 시각화하고 WIP 제한을 통해 병목을 관리하는 작업 관리 기법이다.**

---

## 💬 비유로 이해하기

* **스크럼** 👉 학기제 수업 (기간 고정)
* **칸반** 👉 헬스장 이용 (끝나는 대로 다음)
