# 요구사항 할당 vs 요구사항 협상

## 1️⃣ 요구사항 할당(Requirements Allocation) 예시

### 📌 개념 한 줄

> 전체 시스템 요구사항을 **하위 시스템, 모듈, 컴포넌트에 분배하는 활동**

---

### 🧩 예시: 온라인 쇼핑몰 시스템

#### 🔹 전체 요구사항

* 사용자는 상품을 검색하고 주문할 수 있어야 한다.
* 주문 시 결제가 가능해야 한다.
* 주문 상태를 조회할 수 있어야 한다.

---

#### 🔹 요구사항 할당 과정

| 요구사항     | 할당 대상     |
| -------- | --------- |
| 상품 검색 기능 | 상품 관리 모듈  |
| 주문 생성 기능 | 주문 관리 모듈  |
| 결제 처리 기능 | 결제 모듈     |
| 주문 상태 조회 | 주문 관리 모듈  |
| 결제 승인/취소 | 외부 결제 시스템 |

👉 즉, **“이 요구사항은 누가(어떤 모듈이) 담당할 것인가?”**를 정하는 것

---

### ✍️ 시험용 문장 예시

> 요구사항 할당은 시스템 전체 요구사항을 분석하여 이를 각 서브시스템이나 모듈에 분배하는 과정이다. 예를 들어, 쇼핑몰 시스템에서 결제 기능은 결제 모듈에, 주문 처리는 주문 관리 모듈에 할당한다.

---

## 2️⃣ 요구사항 협상(Requirements Negotiation) 예시

### 📌 개념 한 줄

> 이해관계자 간 **충돌하거나 실현이 어려운 요구사항을 조정·합의하는 활동**

---

### 🧩 예시: 모바일 배달 앱 개발

#### 🔹 고객 요구사항

* 주문 후 **5초 이내**에 배달원 매칭 완료
* 모든 기능을 **2개월 내 개발**

#### 🔹 개발팀 입장

* 실시간 매칭 알고리즘 구현에 최소 4개월 필요
* 인프라 비용과 기술적 한계 존재

---

#### 🔹 요구사항 협상 과정

* 즉시 매칭 → **1차 버전에서는 30초 이내**
* 고급 매칭 알고리즘 → **2차 버전에 반영**
* 핵심 기능 우선 개발 후 단계적 확장

👉 **요구사항을 조정하여 현실적인 범위로 합의**

---

### ✍️ 시험용 문장 예시

> 요구사항 협상은 이해관계자 간 상충되는 요구사항이나 제약 조건을 조정하여 합의하는 과정이다.
> 예를 들어, 개발 기간과 성능 요구가 충돌할 경우 기능 우선순위를 조정하여 단계적으로 구현하기로 협의한다.

---

## 3️⃣ 요구사항 할당 vs 협상 한눈에 비교

| 구분    | 요구사항 할당        | 요구사항 협상         |
| ----- | -------------- | --------------- |
| 목적    | 책임 및 구현 대상 명확화 | 요구사항 충돌 해결      |
| 시점    | 요구사항 분석 후      | 요구사항 분석 중       |
| 대상    | 시스템 내부 구성 요소   | 이해관계자           |
| 핵심 질문 | “누가 구현할 것인가?”  | “어디까지 가능할 것인가?” |
| 결과    | 모듈별 요구사항       | 합의된 요구사항        |

---

## 4️⃣ 시험에서 이렇게 나오면 끝 🔥

* ✔ **할당** → 모듈/서브시스템에 배분
* ✔ **협상** → 충돌 요구사항 조정
* ✔ **할당 ≠ 우선순위**, **협상 ≠ 설계**
