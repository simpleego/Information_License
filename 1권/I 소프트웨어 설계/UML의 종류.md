# UML
- **구조 다이어그램(Structure)**
- **행동 다이어그램(Behavior)** 

---

# 🧱 **1. 구조 다이어그램(Structure Diagrams)**  
시스템이 *어떻게 생겼는지*, 즉 **정적인 구조**를 표현하는 다이어그램들이야.

### ✔ 1) 클래스 다이어그램(Class Diagram)
- 가장 많이 쓰임  
- 클래스, 속성, 메서드, 관계(상속, 연관, 의존 등) 표현  
- 객체지향 설계의 핵심

### ✔ 2) 객체 다이어그램(Object Diagram)
- 클래스 다이어그램의 **실제 인스턴스 상태**를 보여줌  
- 특정 시점의 객체 관계를 표현

### ✔ 3) 컴포넌트 다이어그램(Component Diagram)
- 시스템을 구성하는 **컴포넌트(모듈)** 간 관계 표현  
- 대규모 시스템 구조 파악에 좋음

### ✔ 4) 배치 다이어그램(Deployment Diagram)
- 서버, 장비, 네트워크 등 **물리적 배치 구조** 표현  
- 실제 운영 환경을 시각화

### ✔ 5) 패키지 다이어그램(Package Diagram)
- 클래스들을 패키지 단위로 묶어 구조 표현  
- 대규모 프로젝트 구조 정리

### ✔ 6) 복합 구조 다이어그램(Composite Structure Diagram)
- 객체 내부 구조나 협력 관계 표현  
- 고급 설계에서 사용

# ⚙️ **2. 행동 다이어그램(Behavior Diagrams)**  
시스템이 *어떻게 동작하는지*, 즉 **동적인 행동**을 표현하는 다이어그램들이야.

### ✔ 1) 유스케이스 다이어그램(Use Case Diagram)
- 사용자(Actor)와 시스템 간 상호작용 표현  
- 요구사항 분석 단계에서 가장 많이 사용

### ✔ 2) 시퀀스 다이어그램(Sequence Diagram)
- 객체 간 메시지 흐름을 시간 순서대로 표현  
- API 호출 흐름, 기능 동작 과정 설명에 최적

### ✔ 3) 활동 다이어그램(Activity Diagram)
- 프로세스 흐름도와 비슷  
- 조건, 분기, 반복 등 **업무 흐름** 표현

### ✔ 4) 상태 다이어그램(State Machine Diagram)
- 객체의 **상태 변화**를 표현  
- 예: 주문 상태(주문 → 결제 → 배송 → 완료)

### ✔ 5) 커뮤니케이션 다이어그램(Communication Diagram)
- 시퀀스 다이어그램과 비슷하지만  
- **객체 간 연결 관계**에 초점

### ✔ 6) 상호작용 개요 다이어그램(Interaction Overview Diagram)
- 여러 시퀀스/활동 다이어그램을 묶어 전체 흐름 표현

### ✔ 7) 타이밍 다이어그램(Timing Diagram)
- 시간에 따른 상태 변화 강조  
- 임베디드/하드웨어 설계에서 사용

# 🎯 **전체 UML 종류 요약 표**

| 구분 | 다이어그램 | 설명 |
|------|------------|------|
| **구조** | 클래스 | 정적 구조, 객체지향 설계 핵심 |
| | 객체 | 특정 시점의 객체 상태 |
| | 컴포넌트 | 시스템 구성 요소 관계 |
| | 배치 | 서버/장비 배치 구조 |
| | 패키지 | 패키지 단위 구조 |
| | 복합 구조 | 내부 구조/협력 관계 |
| **행동** | 유스케이스 | 요구사항 분석 |
| | 시퀀스 | 시간 순서 메시지 흐름 |
| | 활동 | 업무/프로세스 흐름 |
| | 상태 | 상태 변화 |
| | 커뮤니케이션 | 객체 간 상호작용 |
| | 상호작용 개요 | 전체 흐름 요약 |
| | 타이밍 | 시간 기반 상태 변화 |

2. **클래스 다이어그램** – 설계  
3. **시퀀스 다이어그램** – 기능 흐름  
4. **활동 다이어그램** – 프로세스 정리
