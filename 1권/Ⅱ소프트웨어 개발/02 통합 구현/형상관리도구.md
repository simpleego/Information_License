# 형상관리 도구
- csv(Concurrent Versions System) 
- svn(Subversion)
- Git(Global Information Tracker)

---

## 📊 CVS, SVN, Git 비교

| 구분 | CVS (Concurrent Versions System) | SVN (Subversion) | Git |
|------|----------------------------------|------------------|-----|
| **방식** | 중앙집중형(CVCS) | 중앙집중형(CVCS) | 분산형(DVCS) |
| **장점** | - 초기 버전 관리 도구로 단순<br>- 기본적인 협업 가능 | - CVS의 단점을 개선<br>- 대규모 바이너리 파일 관리에 유리<br>- 초보자 친화적 | - 분산형 구조로 오프라인 작업 가능<br>- 브랜치/병합이 빠르고 유연<br>- 속도와 성능 우수 |
| **단점** | - 브랜치/병합 기능 미약<br>- 대규모 프로젝트에 부적합<br>- 유지보수 중단 | - 중앙 서버 의존 → 서버 다운 시 전체 작업 중단<br>- 브랜치/병합이 Git보다 느리고 복잡 | - 개념이 복잡해 학습 곡선이 높음<br>- 초보자에게 진입 장벽 |
| **난이도** | 쉬움 (기능 단순) | 중간 (CVS보다 개선, 직관적) | 어려움 (분산형 개념과 다양한 기능 학습 필요) |
| **사용 사례** | 과거 소규모 프로젝트 | 기업 내부 레거시 시스템, 문서 관리 | 현대 소프트웨어 개발 표준, 오픈소스 프로젝트 |
| **개발 시기** | 1990년 | 2000년 | 2005년 |

---

## 🔎 분석
- **CVS**: 가장 오래된 도구로 현재는 거의 사용되지 않음. 단순하지만 기능 부족.  
- **SVN**: CVS의 개선판으로 중앙집중형 관리에 적합. 여전히 일부 기업에서 사용.  
- **Git**: 현재 사실상 표준. 분산형 구조 덕분에 **속도, 유연성, 협업**에서 압도적 우위.  

---

## ⚠️ 고려할 점
- **규모가 작은 프로젝트**나 단순 문서 관리 → SVN이 여전히 실용적일 수 있음.  
- **대규모 협업, 오픈소스, 현대 개발 환경** → Git이 사실상 필수.  
- CVS는 **역사적 의미**만 남아 있으며, 신규 프로젝트에는 적합하지 않음.  

---

👉 결론적으로, **CVS는 과거용, SVN은 중간 단계, Git은 현재와 미래의 표준**이라고 이해하면 됩니다.
