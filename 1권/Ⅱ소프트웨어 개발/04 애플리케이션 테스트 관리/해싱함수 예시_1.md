# 👍 해싱 함수
>  **말로만 보면 추상적인데, 예시 보면 바로 감 잡힘**.
> 아래에서 **같은 키 값**을 써서 하나씩 실제로 어떻게 계산되는지 보여줄게.

---

## 공통 가정

* **키(key)**: `123456`
* **해시 테이블 크기**: `100` (주소 범위 0 ~ 99)

---

## 1️⃣ 제산법 (Division Method)

> 키를 테이블 크기로 나눈 **나머지**를 해시값으로 사용

### 계산

```
h(key) = key % table_size
h(123456) = 123456 % 100 = 56
```

### 결과

👉 **해시 주소 = 56**

📌 가장 많이 쓰이는 기본 방식 (단, 테이블 크기를 잘 골라야 함)

---

## 2️⃣ 제곱법 (Mid-Square Method)

> 키를 제곱한 뒤 **중간 자리**를 사용

### 계산

```
123456² = 15241383936
```

중간 2자리 추출 → `38`

### 결과

👉 **해시 주소 = 38**

📌 키 값의 분포가 골고루 섞이는 장점 있음

---

## 3️⃣ 숫자 분석법 (Digit Analysis Method)

> 키에서 **특정 자리의 숫자들**을 뽑아 조합

### 예시

키: `123456`
뒤에서 2번째와 4번째 자리 선택

```
4번째 자리: 3
2번째 자리: 5
→ 조합: 35
```

### 결과

👉 **해시 주소 = 35**

📌 주민번호, 학번처럼 **자리별 의미가 있는 데이터**에 적합

---

## 4️⃣ 폴딩법 (Folding Method)

> 키를 **일정 자릿수로 나눈 후 더함**

### 계산

```
123 | 456
123 + 456 = 579
```

```
579 % 100 = 79
```

### 결과

👉 **해시 주소 = 79**

📌 긴 키(전화번호, 계좌번호)에 자주 사용

---

## 5️⃣ 기수 변환법 (Radix Transformation)

> 키를 **다른 진수로 변환** 후 해시값 생성

### 예시

```
123456(10진수) → 1E240(16진수)
```

마지막 2자리 사용 → `40(16진수)`

```
40(16) = 64(10)
```

### 결과

👉 **해시 주소 = 64**

📌 문자열 해싱이나 내부 표현 변환에 활용됨

---

## 6️⃣ 무작위법 (Random Method)

> 난수 함수로 해시값 생성

### 예시

```java
Random r = new Random(123456); // 시드값
h(key) = r.nextInt(100);
```

실행 결과 예:

```
h(key) = 72
```

### 결과

👉 **해시 주소 = 72**

📌 보안 목적이나 실험용
📌 일반적인 해시 테이블에는 거의 안 씀

---

## 한눈에 요약 🔍
