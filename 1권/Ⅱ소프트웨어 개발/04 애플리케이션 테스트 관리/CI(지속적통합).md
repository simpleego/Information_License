# CI(Continuous Integration)
---

# 🧩 **CI(Continuous Integration) = “개발자들이 만든 코드를 자주 합치고 자동으로 검사하는 시스템”**


## 🎨 **여러 사람이 함께 그리는 큰 그림을 매일매일 자동으로 맞춰보는 과정**

---

# 🍱 비유로 이해해보자

## 🟡 **상황 1: 여러 사람이 함께 도시락을 만든다**
- A는 반찬 1  
- B는 반찬 2  
- C는 밥  
- D는 국  

각자 따로 만들다가 **마지막에 한 번에 합치면** 어떤 일이 생길까?

- 맛이 안 맞을 수도 있고  
- 어떤 반찬은 너무 짜고  
- 어떤 건 빠져 있고  
- 국물이 새서 도시락이 망가질 수도 있어  

👉 **문제가 생겨도 어디서 잘못됐는지 찾기 어렵다.**

---

# 🟢 **CI는 이런 문제를 막기 위한 자동 검사 시스템**

## 🟡 **상황 2: 반찬을 만들 때마다 즉시 도시락에 넣어보고 자동으로 검사**
- A가 반찬을 만들면 → 자동으로 도시락에 넣어보고 맛 검사  
- B가 반찬을 만들면 → 자동으로 도시락에 넣어보고 조합 검사  
- C가 밥을 넣으면 → 자동으로 전체 구성 검사  

문제가 있으면 바로 알려준다.

👉 **“지금 넣은 반찬이 너무 짜요!”**  
👉 **“밥이 부족해요!”**

이게 바로 **CI(Continuous Integration)**이 하는 일.

---

# 🧠 기술적으로는 이렇게 작동해
- 개발자들이 만든 코드를 **자주(하루 여러 번)** 중앙 저장소에 올리고  
- 올릴 때마다 **자동으로 빌드(조립)**하고  
- **자동으로 테스트**해서  
- 문제가 있으면 즉시 알려주는 시스템

---

# ⭐ CI를 쓰면 좋은 점

### ✔ 문제를 빨리 발견  
작은 문제일 때 바로 잡을 수 있음.

### ✔ 서로의 작업이 충돌하지 않음  
나중에 한꺼번에 합치다가 터지는 일이 줄어듦.

### ✔ 품질이 안정적  
자동 테스트가 계속 돌기 때문에 기본적인 오류는 자동으로 걸러짐.

### ✔ 개발 속도 빨라짐  
사람이 일일이 확인할 필요가 없음.

---

# 🟢 jong을 위한 한 줄 요약  
**CI = 여러 사람이 만든 코드를 자주 합치고 자동으로 검사해서 문제를 빨리 발견하는 시스템.**
