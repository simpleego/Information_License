회복 테스트(Recovery Testing)는 **시스템이 고의적으로 장애나 오류 상황에 빠졌을 때, 얼마나 잘 복구되는지 확인하는 테스트**
쉽게 말하면, “문제가 생겨도 시스템이 다시 정상으로 돌아올 수 있는가?”를 검증하는 과정이지.

---

# 🔧 회복 테스트(Recovery Testing) 예시

## 1) **서버 강제 종료 후 자동 재시작 확인**
- 웹 서버를 실행 중인 상태에서 **전원을 강제로 끄거나 프로세스를 강제 종료(kill)**  
- 다시 켜졌을 때:
  - 서버가 자동으로 재시작되는지  
  - 데이터 손실이 없는지  
  - 사용자 세션이 정상적으로 복구되는지  
  확인하는 테스트

---

## 2) **DB 연결 끊김 상황에서의 복구**
- 애플리케이션이 DB와 통신 중일 때 **DB 서버를 잠시 중단**  
- 이후 DB가 다시 살아났을 때:
  - 애플리케이션이 자동으로 재연결하는지  
  - 오류 메시지가 정상적으로 처리되는지  
  - 트랜잭션이 일관성을 유지하는지  
  확인

---

## 3) **네트워크 장애 후 복구 테스트**
- 네트워크 케이블을 뽑거나 Wi-Fi를 끊어서 **일시적인 네트워크 장애를 발생**  
- 다시 연결했을 때:
  - 요청이 자동 재시도되는지  
  - 앱이 멈추지 않고 정상 동작하는지  
  - 데이터가 중복 저장되지 않는지  
  확인

---

## 4) **파일 시스템 오류 후 복구**
- 파일 저장 중 일부러 디스크 공간을 0으로 만들어 오류 발생  
- 디스크 공간을 확보한 뒤:
  - 저장이 재시도되는지  
  - 손상된 파일이 없는지  
  - 로그가 정상적으로 기록되는지  
  확인

---

## 5) **모바일 앱 강제 종료 후 상태 복구**
- 앱 사용 중 **강제 종료(앱 스와이프 종료)**  
- 다시 실행했을 때:
  - 마지막 화면 상태가 복구되는지  
  - 임시 저장 데이터가 유지되는지  
  - 로그인 세션이 유지되는지  
  확인

---

## 6) **트랜잭션 중단 후 롤백 확인**
- 결제나 주문 처리 중간에 **시스템을 강제로 중단**  
- 재시작 후:
  - 트랜잭션이 정상적으로 롤백되었는지  
  - 중복 결제가 발생하지 않는지  
  확인

---

# 🎯 핵심 요약
회복 테스트는 **장애 → 복구 → 정상 동작 여부**를 검증하는 테스트야.  
특히 금융, 의료, 항공처럼 장애가 치명적인 시스템에서 매우 중요하게 다룬다.
