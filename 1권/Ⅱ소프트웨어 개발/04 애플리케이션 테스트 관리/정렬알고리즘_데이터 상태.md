# 정렬 알고리즘
> “모든 정렬”을 **데이터 상태 기준**으로 분류 비교

---

# 1️⃣ 정렬 알고리즘을 보는 핵심 관점

정렬 선택은 결국 이 질문이야:

> **지금 데이터가 어떤 상태인가?**

* 이미 거의 정렬됨?
* 완전 랜덤?
* 데이터 개수 적음 / 많음?
* 중복 많음?
* 메모리 여유 있음?
* 안정성(Stable) 필요?

---

# 2️⃣ 데이터 상태별 정렬 알고리즘 정리 ⭐ 핵심

## ✅ ① 이미 거의 정렬된 데이터

### 추천 알고리즘

### 🔹 삽입 정렬 (Insertion Sort) ⭐⭐⭐

* **이유**: 이동 횟수 거의 없음
* 시간복잡도: `O(n)`
* 안정 정렬

📌 예:

```
[1, 2, 3, 5, 4, 6]
```

### 🔹 버블 정렬 (개선형)

* 교환 여부 체크 시 빠른 종료
* 실무보다는 교육용

---

## ✅ ② 데이터 개수가 매우 적음 (n ≤ 20)

### 추천 알고리즘

### 🔹 삽입 정렬

### 🔹 선택 정렬

| 알고리즘  | 이유        |
| ----- | --------- |
| 삽입 정렬 | 구현 간단, 빠름 |
| 선택 정렬 | 비교 횟수 고정  |

📌 **n이 작으면 O(n²)도 체감 안 됨**

---

## ✅ ③ 데이터가 완전히 무작위 (일반적인 경우)

### 추천 알고리즘

### 🔹 퀵 정렬 (Quick Sort) ⭐⭐⭐⭐⭐

* 평균 `O(n log n)`
* 캐시 친화적
* 실무 최다 사용

⚠ 단점:

* 피벗 잘못 고르면 `O(n²)`

### 🔹 힙 정렬 (Heap Sort)

* 최악도 `O(n log n)`
* 불안정 정렬

---

## ✅ ④ 최악의 경우 성능이 중요함 (보장 필요)

### 추천 알고리즘

### 🔹 힙 정렬

### 🔹 병합 정렬 (Merge Sort)

| 알고리즘  | 특징        |
| ----- | --------- |
| 힙 정렬  | 메모리 적게 사용 |
| 병합 정렬 | 안정 정렬     |

📌 **실시간 시스템 / 금융 / 서버 로그**

---

## ✅ ⑤ 안정 정렬이 반드시 필요함

> 동일 값의 **기존 순서 유지**

### 추천 알고리즘

* 삽입 정렬
* 병합 정렬
* 버블 정렬
* 계수 정렬
* 기수 정렬

📌 예:

```
(90, 철수), (90, 영희)
```

→ 철수 → 영희 순서 유지

---

## ✅ ⑥ 데이터 범위가 좁고 정수임

### 추천 알고리즘

### 🔹 계수 정렬 (Counting Sort) ⭐⭐⭐⭐

* 시간복잡도: `O(n + k)`
* 비교 기반 아님

📌 예:

```
점수(0~100), 나이, 등급
```

⚠ 메모리 많이 사용

---

## ✅ ⑦ 자릿수 개념이 있는 데이터

### 추천 알고리즘

### 🔹 기수 정렬 (Radix Sort)

* 내부적으로 계수 정렬 사용
* 문자열, 숫자 정렬에 강함

📌 예:

```
전화번호, 주민번호, 우편번호
```

---

## ✅ ⑧ 데이터 분포가 균일함

### 추천 알고리즘

### 🔹 버킷 정렬 (Bucket Sort)

📌 예:

```
0.0 ~ 1.0 실수 데이터
```

---

## ✅ ⑨ 외부 저장장치(디스크) 정렬

### 추천 알고리즘

### 🔹 외부 병합 정렬 (External Merge Sort)

📌 예:

```
수 GB 로그 파일
```

---

# 3️⃣ 한눈에 보는 데이터 상태별 선택표 🔥

| 데이터 상태   | 추천 정렬   |
| -------- | ------- |
| 거의 정렬됨   | 삽입 정렬   |
| 데이터 적음   | 삽입 / 선택 |
| 일반적 무작위  | 퀵 정렬    |
| 최악 성능 중요 | 힙 / 병합  |
| 안정성 필요   | 병합 / 삽입 |
| 범위 작은 정수 | 계수      |
| 자릿수 기반   | 기수      |
| 분포 균일    | 버킷      |
| 외부 저장    | 병합      |

---

# 4️⃣ 시험용 함정 포인트 🚨

* ❌ **퀵 정렬은 항상 빠르다** → ❌ (최악 O(n²))
* ❌ **힙 정렬은 안정 정렬이다** → ❌
* ❌ **비교 정렬만 있다** → ❌ (계수, 기수, 버킷)
* ❌ **병합 정렬은 제자리 정렬이다** → ❌

---

# 5️⃣ 실무적 한 줄 요약 😄

> **대부분의 경우 퀵 정렬,
> 안정성이 필요하면 병합 정렬,
> 거의 정렬됐으면 삽입 정렬**
