# 🧪 1) **API 테스트 스크립트 예시 (REST API)**

```bash
# Test Case: 사용자 로그인 API 테스트

1. 목적:
   - 정상 로그인 요청 시 200 OK와 토큰이 반환되는지 확인한다.

2. 사전 조건:
   - 테스트용 계정 존재 (id: test01, pw: 1234)

3. 요청:
   POST /api/v1/login
   Content-Type: application/json

   {
     "username": "test01",
     "password": "1234"
   }

4. 기대 결과:
   - HTTP Status: 200
   - Response Body:
     {
       "token": "<JWT_TOKEN>",
       "expires_in": 3600
     }

5. 후처리:
   - 반환된 토큰 저장 후 다음 테스트 케이스에서 사용
```

# 🧪 2) **프론트엔드 UI 테스트 스크립트 (Selenium 예시)**

```python
# Test Case: 로그인 버튼 동작 확인

1. 목적:
   - 로그인 페이지에서 올바른 계정 입력 시 대시보드로 이동하는지 확인한다.

2. 절차:
   - 브라우저 실행
   - URL 접속: https://example.com/login
   - ID 입력: test01
   - PW 입력: 1234
   - 로그인 버튼 클릭

3. 기대 결과:
   - URL이 /dashboard 로 변경된다.
   - 화면에 "Welcome test01" 텍스트가 표시된다.
```

# 🧪 4) **QA 시나리오 기반 테스트 스크립트**

```
Test Scenario: 장바구니 담기 기능

1. 목적:
   사용자가 상품을 장바구니에 정상적으로 담을 수 있는지 확인한다.

2. 절차:
   1) 메인 페이지 접속
   2) 상품 목록에서 임의의 상품 선택
   3) "장바구니 담기" 버튼 클릭
   4) 장바구니 페이지 이동

3. 기대 결과:
   - 선택한 상품이 장바구니 목록에 표시된다.
   - 수량 기본값은 1이다.
   - 총 금액이 정확히 계산된다.
```

# 🧪 5) **성능 테스트 스크립트 (JMeter 스타일)**

```
Test Name: 로그인 API 부하 테스트

1. 목적:
   - 1초당 100명의 사용자가 로그인 요청을 보낼 때 서버가 정상 응답하는지 확인한다.

2. 설정:
   - Thread Group: 100 users
   - Ramp-up: 1 second
   - Loop Count: 10

3. 요청:
   POST /api/v1/login

4. 성공 기준:
   - 평균 응답 시간 < 300ms
   - 에러율 < 1%
```
