# 해싱 함수 설계 기법들을 실제 예시
> 각 기법이 **어떻게 동작하는지**를 숫자 예시

---

# 🟡 1) **제산법(Division Method)**  
가장 단순하고 많이 쓰이는 방식.  
**키를 어떤 수로 나눈 나머지를 해시값으로 사용.**

### ✔ 공식  
\[
h(k) = k \mod m
\]

### ✔ 예시  
- 키 \(k = 12345\)  
- 해시 테이블 크기 \(m = 100\)

\[
12345 \mod 100 = 45
\]

👉 **해시값 = 45**

---

# 🟡 2) **제곱법(Mid-Square Method)**  
키를 **제곱한 뒤**, 그 결과의 **중간 부분 숫자**를 해시값으로 사용.

### ✔ 예시  
- 키 \(k = 123\)  
- \(123^2 = 15129\)  
- 중간 숫자 “512”만 추출

👉 **해시값 = 512**

(중간 숫자를 몇 자리 추출할지는 테이블 크기에 따라 결정)

---

# 🟡 3) **숫자 분석법(Digit Analysis Method)**  
키의 **특정 자리수만 선택**해서 해시값으로 사용.  
전화번호, 주민번호처럼 특정 패턴이 있는 데이터에 유용.

### ✔ 예시  
전화번호: **010-3456-7890**

- 앞자리(010)는 모두 같으므로 제외  
- 중간 3456 중 **45**만 사용  
- 뒤 7890 중 **89**만 사용  

👉 선택한 자리수 조합: **4589**

👉 **해시값 = 4589**

---

# 🟡 4) **폴딩법(Folding Method)**  
키를 **여러 조각으로 나누고**, 그 조각들을 **더해서** 해시값을 만든다.

### ✔ 예시  
키: **123456789**

- 3자리씩 나누기 → 123 / 456 / 789  
- 합: 123 + 456 + 789 = **1368**

👉 **해시값 = 1368**

(필요하면 다시 mod 연산으로 테이블 크기에 맞게 줄임)

---

# 🟡 5) **기수 변환법(Radix Conversion Method)**  
키를 **다른 진수로 변환**한 뒤, 그 값을 해시값으로 사용.

### ✔ 예시  
키: **“ABC”** (문자열)  
각 문자 ASCII 코드 → A=65, B=66, C=67  
이를 256진수로 변환한다고 가정하면:

\[
65 \times 256^2 + 66 \times 256 + 67
\]

= 65×65536 + 66×256 + 67  
= 4,259,840 + 16,896 + 67  
= **4,276,803**

👉 **해시값 = 4,276,803**

(보통 마지막에 mod 연산으로 테이블 크기에 맞춤)

---

# 🟡 6) **무작위법(Random Method)**  
키를 입력하면 **난수 생성기(random generator)**를 이용해 해시값을 만든다.  
동일한 키는 항상 동일한 난수를 반환하도록 설계.

### ✔ 예시  
키: **2025**

- 난수 생성기(seed = 2025)  
- 생성된 난수 = **847392**

👉 **해시값 = 847392**

(실제로는 테이블 크기에 맞게 mod 연산)

---

# 🎯 jong을 위한 핵심 요약

| 기법 | 핵심 아이디어 | 실제 예시 |
|------|----------------|-----------|
| 제산법 | 나머지 사용 | 12345 mod 100 = 45 |
| 제곱법 | 제곱 후 중간 숫자 | 123² = 15129 → 512 |
| 숫자 분석법 | 특정 자리수 선택 | 전화번호 3456-7890 → 4589 |
| 폴딩법 | 조각내서 더함 | 123/456/789 → 1368 |
| 기수 변환법 | 다른 진수로 변환 | “ABC” → 4,276,803 |
| 무작위법 | 난수 생성기 사용 | seed=2025 → 847392 |
