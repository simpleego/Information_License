#  해싱함수 문자열_충돌 예시
> 해싱함수에서  **문자열 해싱 실제 예제 → 충돌 발생 → 해결 방식 비교**까지 한 번에 정리

---

# 1️⃣ 문자열 해싱 기본 예제

## 예제 문자열

* `"CAT"`
* `"ACT"`
* `"DOG"`

## 가정

* 문자 → 아스키 코드 사용
* 해시 테이블 크기: `10`

---

## 방법 1: 단순 합 방식 (가장 흔한 예제)

### 해시 함수

```
h(key) = (문자 ASCII 값의 합) % 10
```

### 계산

### 🔹 "CAT"

```
C = 67
A = 65
T = 84
합 = 216
h = 216 % 10 = 6
```

### 🔹 "ACT"

```
A = 65
C = 67
T = 84
합 = 216
h = 216 % 10 = 6
```

### 🔹 "DOG"

```
D = 68
O = 79
G = 71
합 = 218
h = 218 % 10 = 8
```

### 결과

| 문자열 | 해시값 |
| --- | --- |
| CAT | 6   |
| ACT | 6 ❗ |
| DOG | 8   |

👉 **CAT과 ACT 충돌 발생**

📌 이유:
순서가 달라도 **문자 합이 같으면 같은 해시값**

---

# 2️⃣ 충돌 줄이기: 가중치(위치 반영) 방식

## 해시 함수

```
h(key) = (Σ ASCII × 위치 가중치) % 10
```

(왼쪽부터 1, 2, 3…)

---

### 🔹 "CAT"

```
C(67)×1 + A(65)×2 + T(84)×3
= 67 + 130 + 252 = 449
449 % 10 = 9
```

### 🔹 "ACT"

```
A(65)×1 + C(67)×2 + T(84)×3
= 65 + 134 + 252 = 451
451 % 10 = 1
```

### 결과

| 문자열 | 해시값 |
| --- | --- |
| CAT | 9   |
| ACT | 1   |

✅ **충돌 해결**

📌 실무 해싱의 기본 아이디어
→ **문자 순서를 반드시 반영**

---

# 3️⃣ 다항 해싱 (Polynomial Hashing) ⭐ 실무/시험 핵심

## 해시 함수

```
h(key) = (Σ 문자값 × pⁱ) % m
```

* p: 보통 31, 33, 37
* m: 해시 테이블 크기

---

### 예제

* p = 31
* m = 10

### 🔹 "CAT"

```
67×31² + 65×31¹ + 84×31⁰
= 67×961 + 65×31 + 84
= 64487 + 2015 + 84
= 66586
66586 % 10 = 6
```

### 🔹 "ACT"

```
65×31² + 67×31 + 84
= 62465 + 2077 + 84
= 64626
64626 % 10 = 6 ❗
```

👉 **테이블이 너무 작으면 여전히 충돌**

📌 테이블 크기 중요성 강조 포인트

---

# 4️⃣ 충돌 해결 방법 비교

## ① 체이닝 (Chaining)

```
[6] → CAT → ACT
```

### 특징

* 연결 리스트 사용
* 구현 쉬움
* 테이블 꽉 차도 동작 가능

📌 **시험에서 가장 많이 등장**

---

## ② 선형 조사법 (Linear Probing)

```
h = 6
6 → CAT
7 → ACT
```

### 특징

* 빈 칸 찾을 때까지 +1
* **1차 군집화 문제 발생**

---

## ③ 이차 조사법 (Quadratic Probing)

```
h + 1², 2², 3² …
6 → 7 → 10 → ...
```

### 특징

* 군집화 완화
* 구현은 조금 복잡

---

## ④ 이중 해싱 (Double Hashing) ⭐ 고급

```
h1 = key % m
h2 = c - (key % c)
```

### 특징

* 충돌 분산 효과 최고
* 성능 가장 좋음

---

# 5️⃣ 충돌 상황 한눈에 비교

| 방법    | 충돌 처리    | 장점     | 단점         |
| ----- | -------- | ------ | ---------- |
| 체이닝   | 리스트 연결   | 구현 쉬움  | 포인터 사용     |
| 선형 조사 | +1 이동    | 간단     | 군집화        |
| 이차 조사 | 제곱 이동    | 군집화 감소 | 빈칸 못 찾을 수도 |
| 이중 해싱 | 다른 해시 사용 | 성능 우수  | 복잡         |

---

# 6️⃣ 정보처리기사 시험용 핵심 암기 💡

* **문자열 단순 합 → 충돌 많음**
* **문자 위치 반영 → 충돌 감소**
* **충돌 해결 = 체이닝, 개방 주소법**
* **개방 주소법 종류**
  → 선형 / 이차 / 이중 해싱
* **해시 성능 좌우 = 테이블 크기 + 해시 함수**
