# **정보처리기사 소프트웨어 개발 과목 요약
---

## 📌 핵심 학습 영역 (4시간 전략)

### 1. 통합 구현 및 빌드 (1시간)
- **통합 구현 원리**: 분할과 집합, 정보 은닉, 자료 추상화, 모듈 독립성  
- **구현 단계**: 계획 → 코딩 → 컴파일 → 테스트 → 단위 모듈 테스트  
- **빌드 도구**: Ant, Maven, Gradle 특징 비교  

👉 **모듈 독립성과 빌드 도구 특징**은 시험 단골 문제.

---

### 2. 테스트 기법 (1시간)
- **블랙박스 테스트**: 동등 분할, 경계값 분석, 원인-결과 그래프, 오류 예측  
- **화이트박스 테스트**: 구문/조건/경로 커버리지  
- **테스트 단계**: 단위 → 통합 → 시스템 → 인수 테스트  

👉 **블랙박스 vs 화이트박스 차이**는 반드시 암기.

---

### 3. 형상 관리 (30분)
- **형상 관리 도구**: Git, SVN, CVS 특징  
- **버전 관리 개념**: 체크아웃, 커밋, 브랜치, 머지  
- **빌드 자동화와 CI/CD**: Jenkins, GitHub Actions  

👉 **Git 명령어와 형상 관리 개념**은 기출에서 반복 출제.

---

### 4. 알고리즘 & 자료구조 (1시간)
- **정렬 알고리즘**: 버블, 선택, 삽입, 퀵, 병합 → 시간복잡도 비교  
- **탐색 알고리즘**: 선형 탐색, 이진 탐색  
- **자료구조**: 스택, 큐, 연결리스트, 트리, 그래프  
- **알고리즘 설계 기법**: 분할정복, 동적계획법, 탐욕 알고리즘  

👉 **시간복잡도와 알고리즘 특징**은 계산 문제로 자주 출제.

---

### 5. 프로그래밍 언어 활용 (30분)
- **C/Java/Python 기본 문법**: 제어문, 함수, 클래스, 예외 처리  
- **SQL 기본 문법**: SELECT, JOIN, GROUP BY, 서브쿼리  
- **스크립트 언어**: JavaScript, Python 특징  

👉 **SQL과 Java 기본 문법**은 실무형 문제 대비 필수.

---

## ⚠️ 시험 대비 팁
- **기출문제 병행 학습**: 단순 암기보다 문제 유형 파악이 중요.  
- **출제 비중 높은 영역**: 통합 구현, 테스트, 알고리즘.  
- **암기형 문제 대비**: 빌드 도구, 형상 관리, SQL 구문은 키워드 중심으로 정리.
