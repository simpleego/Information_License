# SW개발방법론 이해
---

## 1️⃣ 폭포수 모델 (Waterfall Model)

### 📌 비유: **집 짓기 🏠**

**과정**

1. 설계도 작성
2. 기초 공사
3. 골조 공사
4. 내부 공사
5. 입주

**왜 비슷한가?**

* 이전 단계가 끝나야 다음 단계 진행
* 중간에 “방 하나 더 만들자” 하면 큰 문제 발생

**핵심 이해**

> 요구사항이 처음부터 명확해야 하는 모델

---

## 2️⃣ 프로토타입 모델 (Prototype Model)

### 📌 비유: **옷 맞춤 제작 👔**

**과정**

1. 샘플 옷 제작
2. 입어보고 피드백
3. 수정
4. 최종 완성

**왜 비슷한가?**

* 완성 전 미리 써봄
* 고객 의견을 계속 반영

**핵심 이해**

> 사용자가 원하는 걸 **보면서 구체화**하는 모델

---

## 3️⃣ 반복적 모델 (Iterative Model)

### 📌 비유: **요리 레시피 개선 🍳**

**과정**

1. 1차 요리
2. 맛 평가
3. 재료/조리법 수정
4. 다시 요리

**왜 비슷한가?**

* 완벽하지 않아도 계속 개선
* 점점 더 나아짐

**핵심 이해**

> 개발 → 평가 → 개선을 반복

---

## 4️⃣ 점증적 모델 (Incremental Model)

### 📌 비유: **스마트폰 기능 업데이트 📱**

**과정**

* 1단계: 전화, 문자
* 2단계: 카메라 추가
* 3단계: 앱, 결제 기능

**왜 비슷한가?**

* 기능을 하나씩 추가
* 매번 사용 가능한 결과물 제공

**핵심 이해**

> 기능을 **조금씩 나눠서 완성**

---

## 5️⃣ 애자일 모델 (Agile Model)

### 📌 비유: **배달 음식 주문 🍔**

**과정**

* 고객 주문
* 빠른 조리
* 피드백
* 다음 주문 반영

**왜 비슷한가?**

* 계획보다 변화 대응이 중요
* 고객과 지속적 소통

**핵심 이해**

> 변화에 유연하고 빠른 개발 방식

---

## 6️⃣ 나선형 모델 (Spiral Model)

### 📌 비유: **신제품 자동차 개발 🚗**

**과정**

1. 계획 수립
2. 위험 분석
3. 개발
4. 고객 평가
   → 반복

**왜 비슷한가?**

* 위험 요소를 먼저 검토
* 고위험 프로젝트에 적합

**핵심 이해**

> 위험 관리 중심의 반복 모델

---

## 7️⃣ V-모델

### 📌 비유: **시험 공부와 채점 📚**

**과정**

* 공부 계획 ↔ 시험 출제
* 단원 학습 ↔ 단원 평가
* 전체 학습 ↔ 종합 평가

**왜 비슷한가?**

* 개발 단계마다 검증 단계 존재

**핵심 이해**

> 개발과 테스트를 1:1로 대응

---

## 8️⃣ 빅뱅 모델 (Big Bang Model)

### 📌 비유: **즉흥 요리 🍜**

**과정**

* 계획 없이 요리
* 재료 넣고 즉석 완성

**왜 비슷한가?**

* 계획 부족
* 결과 예측 어려움

**핵심 이해**

> 소규모·실험적 프로젝트에만 적합

---
